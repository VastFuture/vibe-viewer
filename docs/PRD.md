> 生成时间：2026-04-30

## 1. 一句话定义

vibe-viewer 是一个面向开发者和技术写作者的本地 Markdown 浏览器，通过一行命令在浏览器中浏览、搜索和实时预览指定目录下的所有 Markdown 文件，核心解决「本地 Markdown 文件缺少丝滑阅读体验」的问题。

## 2. 用户流程与场景

### 用户画像

- **主要用户**：开发者、技术写作者、知识管理爱好者
- **技术水平**：熟悉命令行，能在终端执行 `npx` 命令
- **使用频率**：日常工作中频繁查看文档（README、设计文档、API 文档、笔记库）
- **设备**：桌面端为主（macOS / Linux / Windows），偶尔移动端查看

### 核心任务流

```
npx vibe-viewer → 浏览器自动打开 → 左侧选文件 → 右侧看渲染结果
    ↓                                          ↓
  指定目录                                   搜索过滤
    ↓                                          ↓
  交互选择                                   全屏查看图表
```

1. **启动**：终端执行 `npx vibe-viewer` 或 `npx vibe-viewer --dir /path`
2. **浏览**：在文件树中点击或搜索目标文件
3. **阅读**：右侧渲染区实时预览 Markdown 内容
4. **交互**：点击 Mermaid 图表进入全屏查看（缩放/平移/拖拽）
5. **实时更新**：修改文件后浏览器自动刷新预览

### 入口场景

| 场景 | 入口 | 行为 |
|------|------|------|
| 快速查看项目 README | `npx vibe-viewer` 在项目根目录 | 自动打开当前目录的 Markdown 文件 |
| 浏览笔记库 | `npx vibe-viewer --dir ~/notes` | 指定目录，浏览所有笔记 |
| 阅读设计文档 | 交互式选择目录 | 弹出目录列表，选择目标目录 |

## 3. 视觉风格

- **调性**：极简专业，暗色优先，不抢内容的风头
- **主色**：由主题系统决定（8 套主题），默认 GitHub Dark
- **参照**：VS Code 侧边栏 + GitHub Markdown 渲染风格
- **主题**：深色为主（6 深色 + 2 浅色），运行时切换无闪烁
- **圆角**：大圆角（14px 面板，12px 卡片，10px 按钮/输入框）
- **背景**：带渐变光晕的半透明面板（radial-gradient + backdrop-filter: blur）

### 侧边栏折叠

- 展开时：侧边栏右上角 `◀` 按钮，点击收起
- 收起时：左上角浮动 `▶` 按钮，点击展开
- 折叠使用 CSS `visibility: hidden` + `opacity: 0`，不做 DOM 卸载

### 移动端适配

- 860px 以下：单栏布局（侧边栏堆叠在预览区上方）
- 需要增强：移动端应支持侧边栏切换（当前仅是堆叠）

## 5. 交互说明

### 关键交互

| 触发 | 行为 | 反馈 | 异常处理 |
|------|------|------|----------|
| 点击 Mermaid 图表 | 打开全屏查看器 | 浮层淡入动画 | 无 Mermaid 节点时无反应 |
| 全屏查看器 - 滚轮 | 阶梯式缩放 | 显示百分比标签 | 限制 20%-500% 范围 |
| 全屏查看器 - 拖拽 | 无限画布平移 | 拖拽光标变化 | `setPointerCapture` 防丢帧 |
| 全屏查看器 - Esc/✕ | 关闭浮层 | 淡出 | — |
| 侧边栏 - ◀ 按钮 | 收起侧边栏 | Grid 动画过渡 | — |
| 侧边栏 - ▶ 按钮 | 展开侧边栏 | Grid 动画过渡 | — |
| 文件树 - 点击文件 | 加载 Markdown | 路径 + mtime 更新 | 加载失败显示错误提示 |
| 文件树 - 过滤输入 | 实时过滤文件名 | 匹配文件高亮 | 无匹配时列表为空 |
| 设置 - 切换主题 | 即时切换 CSS 变量 | 全局无闪烁切换 | — |
| WebSocket 断连 | 状态提示更新 | "WS: 已断开（自动重连）" | 自动重连 |
| 文件保存 | 自动刷新预览 | 内容静默更新 | 多文件变更只刷一次树 |

### 状态定义

- **文件树状态**：加载中 → 有数据 → 空目录 → 加载失败
- **预览状态**：已选文件 → 渲染中 → 渲染完成 → Mermaid 渲染中/失败
- **WebSocket 状态**：连接中 → 已连接 → 已断开（自动重连）
- **全屏查看器状态**：关闭 → 打开（缩放中/平移中）

## 6. 数据逻辑

### 核心实体

| 实体 | 关键字段 | 关系 |
|------|----------|------|
| FileTree | items: TreeItem[], root: string | 包含多个 TreeItem |
| TreeItem | name, type(file/dir), absPath, children?, mtimeMs? | dir 可嵌套 TreeItem |
| MarkdownFile | absPath, relPath, content, mtimeMs | 对应文件系统中的 .md 文件 |
| Settings | theme, securityLevel, breaks, emoji, math | 持久化到 localStorage |

### 接口需求

| 页面 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 主界面 | /api/config | GET | 返回 rootAbs, rootName, extensions |
| 主界面 | /api/tree | GET | 递归扫描目录，返回 FileTree |
| 预览区 | /api/file?path= | GET | 读取 Markdown 文件内容 |
| 资源 | /raw?path= | GET | 读取图片等静态资源 |

### 数据流向

```
文件系统 ──chokidar──▶ WebSocket ──▶ React State ──▶ UI 重渲染
     │                                        │
     └── /api/tree, /api/file ───────────────┘
```

- **全局数据**：config、tree、current、settings、wsStatus
- **持久化**：Settings + lastFile → localStorage
- **实时同步**：WebSocket 推送 tree-changed / file-changed 事件

### 安全边界

- `/raw` 端点需验证路径在 root 目录内（当前未做，是已知风险）
- DOMPurify 在 `allow-html` 模式下净化输出
- `strict` 模式下 markdown-it 禁用 HTML 标签

## 7. 运营设计

### 增长路径

vibe-viewer 是开源本地工具，增长主要靠：

1. **拉新**：npm 下载量、GitHub README、开发者社区传播

## 8. 假设与待确认项

### 已确认假设

- **[假设]** 目标用户是命令行用户，不需要 GUI 目录选择器（CLI 交互式选择足够）
- **[假设]** 默认主题 GitHub Dark 符合大多数开发者偏好
- **[假设]** Mermaid 全屏查看器使用 `<img> + Blob URL` 隔离 CSS 上下文是正确方案
- **[假设]** 全屏查看器的无限画布拖拽（CSS transform + translate）比滚动方案体验更好
- **[假设]** 8 个主题足以覆盖主流偏好，暂不做主题自定义
- **[假设]** 本地工具场景下 `/raw` 路径不校验 root 目录可接受

### 待确认项

1. **移动端交互**：当前 860px 断点仅做列堆叠，是否需要抽屉式侧边栏？
2. **导出功能**：是否需要「导出为 PDF / HTML」功能？
3. **多标签页**：是否需要同时打开多个文件标签页？
4. **搜索全文**：当前只支持文件名过滤，是否需要全文搜索？
5. **目录书签**：是否需要记住常用目录列表？
6. **`/raw` 路径安全**：是否需要在服务端校验请求路径在 root 目录内？
