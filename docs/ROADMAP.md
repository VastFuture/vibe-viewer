## 8. 预案

### 预案：Mermaid 全屏查看器渲染异常

**触发条件**：Epic 1 第 3 天仍有 2+ 种图表类型渲染不正确
**Plan A（首选）**：`<img> + Blob URL` 方案，逐个修复 CSS 变量替换
**Plan B（降级）**：回退到内联 SVG + `width: 100%` 方案，牺牲 CSS 隔离性
**Plan C（兜底）**：全屏查看器改为新窗口打开，用户可浏览器原生缩放
**决策时间点**：第 1 周周五

### 预案：移动端手势兼容性问题

**触发条件**：Epic 4 第 2 天触摸事件在 iOS Safari 上不流畅
**Plan A（首选）**：Pointer Events + setPointerCapture
**Plan B（降级）**：Touch Events + 防抖处理
**Plan C（兜底）**：移动端仅支持按钮缩放，不支持手势拖拽
**决策时间点**：第 2 周周三

### 预案：6 周产能不足

**触发条件**：第 4 周末发现累计延期 > 3 天
**Plan A**：砍 Epic 5（状态 UI），只做最关键的加载状态
**Plan B**：砍 Epic 6（测试文档），推迟到 Q3
**决策人**：fenghaolin
